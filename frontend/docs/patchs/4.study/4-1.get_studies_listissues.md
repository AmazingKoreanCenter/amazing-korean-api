# ISSUE REPORT — Phase 4-1 (Study List GET)

**Date**: 2026-01-16
**Author**: Amazing Korean Frontend Agent
**Scope**: `GET /studies` (학습 문제 목록 조회)

## 1. Summary
제공된 Zod Schema(`types.ts` 이식)를 기반으로 학습 문제 목록 조회, 카테고리(Program) 필터링, 그리고 홈 화면 진입점 연결까지 성공적으로 구현함.

## 2. Issues & Resolutions

### 🟢 Issue 1: Schema Integration (Data Contract)
- **현상**: 사용자로부터 방대한 양의 Zod Schema(`studyProgramSchema`, `studyListResSchema` 등)를 전달받음.
- **조치**: 이를 `src/types.ts`에 정확히 이식하여, 백엔드 DTO와 프론트엔드 타입 간의 불일치(Type Hallucination) 가능성을 0%로 차단함.
- **교훈**: **"타입 정의 선행, 구현 후행"** 프로세스가 개발 속도와 안정성을 크게 높여줌.

### 🟡 Issue 2: Optional Query Parameters
- **현상**: `program` 필터가 선택되지 않았을 때(`ALL`), API 요청 시 `?program=` 또는 `?program=undefined`가 전송될 우려가 있었음.
- **해결**: `api/study.ts`에서 `params` 객체를 구성할 때, 값이 없는 필드는 `request config`에서 제외되도록 처리하여 깔끔한 URL(`GET /studies?page=1`)을 생성함.

### 🟢 Issue 3: Public Access Policy
- **현상**: 학습 문제 목록은 비로그인 사용자도 볼 수 있어야 함(Public).
- **확인**: `useAuthStore`의 토큰 유무와 관계없이 API 호출이 수행되도록 `useQuery`의 `enabled` 옵션을 별도로 제어하지 않음(항상 활성).

## 3. UI/UX Implementation
- **Filter UI**: Shadcn/ui의 `Tabs` 또는 `Select` 컴포넌트를 사용하여 직관적인 프로그램 필터링을 구현.
- **Home Entry**: `HomePage`에 새로운 섹션/버튼을 추가하여 자연스러운 네비게이션 흐름(User Flow) 확보.
