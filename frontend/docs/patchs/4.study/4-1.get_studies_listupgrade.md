# UPGRADE PROPOSAL — Phase 4-1 (Study List GET)

**Date**: 2026-01-16
**Author**: Amazing Korean Frontend Agent
**Scope**: Study List UX & Data Richness

## 1. Overview
현재 구현된 `GET /studies`는 기본적인 페이징과 카테고리(Program) 필터링만 제공합니다.
하지만 학습 플랫폼의 핵심은 **"내가 푼 문제와 안 푼 문제 구분"** 과 **"나에게 맞는 난이도/주제 찾기"** 입니다. 이를 위해 목록 조회 API의 응답 데이터를 풍부하게 만들고 UI 기능을 강화해야 합니다.

## 2. Proposed Features

### 🎯 Priority A: Progress Indicators (Solved Badge)
- **Problem**: 사용자는 목록만 보고 내가 이 문제를 이미 풀었는지, 점수는 몇 점이었는지 알 수 없어 중복 클릭이 발생함.
- **Solution**:
    - **Backend**: `GET /studies` 응답 DTO(`StudyListItem`)에 로그인한 유저 기준으로 `is_solved: boolean`, `best_score: number` 필드 추가 (Join Query 필요).
    - **Frontend**:
        - 푼 문제(Solved): 카드 우측 상단에 **초록색 '풀이 완료(Solved)' 배지**와 **최고 점수** 표시.
        - 안 푼 문제: 'New' 또는 일반 상태 유지.
    - **Effect**: 성취감 고취 및 학습 동기 부여 (리스트를 초록색으로 채우고 싶은 심리 자극).

### 🔍 Priority B: Keyword Search (Search Bar)
- **Problem**: 특정 문법이나 단어(예: "은/는", "가족")가 포함된 문제를 찾기 어려움.
- **Solution**:
    - **Backend**: `q` (Query) 파라미터 추가 → `title`, `subtitle` 대상 `ILIKE` 검색.
    - **Frontend**: 필터 탭 우측에 돋보기 아이콘과 검색어 입력창(`Input`) 배치. Debounce(300ms) 처리하여 실시간 검색 지원.

### 🏷️ Priority C: Difficulty & Tags (Meta Info)
- **Problem**: "초급용"인지 "고급용"인지 제목만 보고 판단하기 어려움.
- **Solution**:
    - **Data**: 각 Study 데이터에 `difficulty` (1~5) 또는 `tags` (["문법", "듣기"]) 메타데이터 추가.
    - **UI**: 카드 하단에 난이도(별점 ⭐⭐⭐)나 태그 칩(Chip) 노출.

### 📱 Priority D: Infinite Scroll (Mobile UX)
- **Current**: 숫자 기반 페이지네이션 (1, 2, 3...). 데스크톱 친화적.
- **Upgrade**: 모바일에서는 스크롤이 바닥에 닿으면 다음 페이지를 로드하는 **무한 스크롤(Infinite Scroll)** 이 훨씬 자연스러움.
    - **Tech**: `IntersectionObserver` 또는 React Query의 `useInfiniteQuery` 도입 고려.
