# PATCH REQUEST — FRONTEND Phase 1-5 Find ID UI

**ROLE**:
- Frontend Developer (React, Zod, Shadcn/ui, React Query).

**OBJECTIVE**:
- 회원 아이디 찾기 페이지(`/find-id`)를 구현하고, 백엔드 API(`POST /auth/find-id`)와 연동하시오.
- **보안 정책(Enumeration Safe)**을 준수하여, 사용자가 존재하든 하지 않든 동일한 성공 메시지를 표시해야 합니다.

**MCP ACTIONS**:
1.  **Read Types**: `src/category/user/types.ts` 또는 `auth/types.ts`를 확인하여 요청 DTO(`name`, `email`) 구조를 파악하시오. (없다면 `FindIdReq` 타입을 새로 정의)
2.  **Read Components**: `src/components/ui` 폴더의 Input, Button, Card 등을 확인.

**UI SPEC (Shadcn First)**:
- **Path**: `/find-id`
- **Layout**: `max-w-md` 중앙 정렬 Card (로그인 페이지와 통일감 유지).
- **Components**:
    - `<Form>` (React Hook Form + Zod).
    - `<Input>`: 이름(Name), 이메일(Email).
    - `<Button>`: "아이디 찾기" (Loading 상태 표시).
    - **Links**: "로그인으로 돌아가기", "비밀번호 찾기".
- **Feedback (State Machine)**:
    - **Success (200)**: Toast 또는 Alert로 "입력하신 정보가 일치할 경우, 등록된 이메일로 아이디가 발송됩니다." 표시. (절대 가입 여부를 노출하지 말 것).
    - **Rate Limit (429)**: Toast "너무 많은 시도가 감지되었습니다. 잠시 후 다시 시도해주세요."

**DATA SPEC**:
- **API Endpoint**: `POST /api/auth/find-id`
- **Request Body**: `{ "name": "...", "email": "..." }`
- **Response**:
    - **200 OK**: 성공 (본문 무시).
    - **400 Bad Request**: 형식 오류.
    - **429 Too Many Requests**: 레이트 리밋.

**IMPLEMENTATION STEPS**:
1.  **Type Definition**: `src/category/auth/types.ts`에 `FindIdReq` 추가.
2.  **API Function**: `src/category/auth/auth_api.ts` -> `findId(data: FindIdReq)` 추가.
3.  **Hook**: `src/category/auth/hook/use_find_id.ts` -> `useMutation`.
4.  **Component**: `src/category/auth/page/find_id_page.tsx` 구현.
