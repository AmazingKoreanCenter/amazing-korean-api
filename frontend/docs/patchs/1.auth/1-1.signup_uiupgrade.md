# 🚀 Backlog: Signup UI Upgrade Plan (Phase 1-1+)

## 📋 개요
Phase 1-1(MVP) 회원가입 기능의 **보안성(Security), 사용자 경험(UX), 국제화(i18n)**를 강화하기 위한 고도화 과제 목록입니다.
이 작업들은 핵심 기능(로그인, 비디오 학습) 구현이 완료된 후 순차적으로 진행합니다.

---

## 1. 보안 및 검증 강화 (Security & Verification)

### ✅ 이메일 소유권 검증 (Email Verification)
- **목적**: 무분별한 가입 방지 및 실제 사용자 식별.
- **현재**: 이메일 형식만 체크 후 즉시 가입 완료.
- **개선안**:
  1. 회원가입 폼 제출 시 상태를 `pending`으로 저장.
  2. 백엔드에서 인증 코드(6자리) 또는 링크가 담긴 이메일 발송.
  3. 프론트엔드에 **"인증 코드 입력 모달"** 노출.
  4. 코드 일치 시 계정 상태를 `active`로 변경하고 토큰 발급.
- **Tech**: `Nodemailer` (Backend), `InputOTP` (Shadcn UI Component).

### ✅ 접속 환경 및 로그 정밀 기록 (Audit Logging)
- **목적**: 계정 도용 시 추적 및 부정 사용 감지.
- **현재**: 기본적인 User Agent 문자열만 전송.
- **개선안**:
  1. **Device Fingerprinting**: 프론트에서 `ua-parser-js` 등을 사용해 OS, 브라우저 버전, 디바이스 종류(Mobile/Desktop)를 정밀하게 파싱.
  2. **Geo-Location**: 접속 IP를 기반으로 대략적인 위치 파악 (Backend).
  3. 해당 정보를 `USERS_LOG` 테이블에 구조화된 JSON으로 저장.

### ✅ 국가 코드와 IP 불일치 감지 (Fraud Detection)
- **목적**: VPN 사용이나 국적 위조 의심 계정 모니터링.
- **로직**:
  - 사용자 입력 국가(예: KR) vs 접속 IP 기반 국가(예: RU) 비교.
  - **Action**: 불일치 시 `users` 테이블에 `risk_level: high` 플래그 설정 또는 추가 인증 요구.

## 2. 글로벌 UX 최적화 (Internationalization)

### 📅 생년월일 입력 포맷 유연화 (Localized Date Picker)
- **목적**: 국가별 날짜 표기법 차이(YYYY-MM-DD vs MM/DD/YYYY)로 인한 입력 오류 방지.
- **현재**: 단순 텍스트 입력 또는 고정된 포맷.
- **개선안**:
  - **Calendar UI**: 텍스트 입력 대신 Shadcn `Calendar` + `Popover` 컴포넌트 사용.
  - **Locale Support**: 사용자의 브라우저 로케일에 맞춰 달력 및 날짜 표시 순서 자동 변경 (`date-fns` 활용).
  - 전송 시에는 항상 표준인 `YYYY-MM-DD`로 변환하여 백엔드 전송.

### 🌐 언어 선택의 자국어 표기 (Native Language Names)
- **목적**: 사용자가 자신의 언어를 빠르게 찾을 수 있도록 배려.
- **현재**: 영어 또는 한국어로만 표기될 가능성 있음.
- **개선안**:
  - 드롭다운 메뉴에 해당 언어의 **원어(Native Name)**를 표시.
  - 예:
    - `Korean` -> **한국어**
    - `English` -> **English**
    - `Japanese` -> **日本語**
  - **Tech**: `Intl.DisplayNames` API 활용 또는 정적 매핑 테이블 사용.
