# PATCH REQUEST — FRONTEND Phase 1-1 Signup UI

**ROLE**:
- Frontend Developer (React, Zod, Shadcn/ui, Zustand).

**OBJECTIVE**:
- 회원가입 페이지(`/signup`)를 구현하고, 백엔드 API(`POST /users`)와 연동하시오.
- **Vertical Slicing**: 모든 코드는 `src/category/auth/` 폴더 내에 위치해야 합니다.

**MCP ACTIONS**:
1.  **Read Types**: `src/category/user/types.ts`의 `SignupReq`, `SignupRes` 구조를 확인하시오. (snake_case 필수)
2.  **Read Store**: `src/hooks/use_auth_store.ts`를 확인하여, 가입 성공 시 로그인 처리(`login` 액션) 가능 여부를 파악하시오.
3.  **Read API Client**: `src/api/client.ts`를 참조하여 API 요청 패턴을 확인하시오.

**UI SPEC (Shadcn First)**:
- **Path**: `/signup`
- **Layout**: `max-w-md` 중앙 정렬 Card.
- **Components**:
    - `<Form>` (React Hook Form + Zod Resolver)
    - `<Input>`: Email, Password, Name, Nickname, Birthday(YYYY-MM-DD)
    - `<Select>`: Gender(male/female/other), Country(KR/US/...), Language(ko/en/...)
    - `<Checkbox>`: 이용약관(terms_service), 개인정보(terms_personal) - **필수 체크**
    - `<Button>`: "가입하기" (Loading 상태 시 Spinner 표시)
- **Feedback**:
    - Validation Error: 각 필드 아래에 붉은색 텍스트로 즉시 표시.
    - API Error: Toast로 에러 메시지 표시 (예: "이미 사용 중인 이메일입니다.").
    - Success: Toast "가입을 환영합니다!" 표시 후 메인 페이지(`/`)로 이동.

**DATA SPEC (Strict snake_case)**:
- **API Endpoint**: `POST /api/users` (Proxy 적용을 위해 `/api` 접두사 필수)
- **Request DTO**: `src/category/user/types.ts`의 `SignupReq` 사용.
    - **주의**: `birthday` (O), `birthDay` (X) / `user_state` (O)
- **Validation Rules**:
    - Password: min 8, max 72
    - Nickname: min 1, max 100
    - Terms: must be true
    - Birthday: Valid date string (YYYY-MM-DD)

**IMPLEMENTATION STEPS**:
1.  **API Function**: `src/category/auth/auth_api.ts` 생성 -> `signup(data: SignupReq)` 함수 구현.
2.  **Hook**: `src/category/auth/hook/use_signup.ts` -> `useMutation` 사용. 성공 시 `useAuthStore.login(data)` 호출.
3.  **Component**: `src/category/auth/page/signup_page.tsx` -> UI 및 폼 로직 구현.
4.  **Route**: `src/app/routes.tsx` (또는 `App.tsx`)에 라우트 추가.
