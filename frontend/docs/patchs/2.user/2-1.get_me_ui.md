# PATCH REQUEST — FRONTEND Phase 2-1 User Me UI

**ROLE**:
- Frontend Developer (React, React Query, Shadcn/ui, TypeScript).

**OBJECTIVE**:
- 로그인한 사용자의 상세 정보를 조회하는 '마이 페이지(`/user/me`)'를 구현하시오.
- `GET /users/me` API를 연동하고, `HomePage`에 있는 '마이 페이지' 버튼이 정상 작동하도록 연결하시오.

**MCP ACTIONS**:
1.  **Read Types**: `src/category/auth/types.ts`를 확인하여 현재 유저 타입의 한계를 파악.
2.  **Read Home**: `src/category/home/page/home_page.tsx`를 읽어 네비게이션 버튼 경로(`/user/me`)를 확인.

**UI SPEC (Shadcn First)**:
- **Path**: `/user/me`
- **Layout**: `max-w-lg` 중앙 정렬 Card.
- **Components**:
    - `<Card>`: 회원 정보 컨테이너.
    - `<Avatar>`: 프로필 이미지 (이메일 첫 글자 등으로 Fallback).
    - `<Skeleton>`: 데이터 로딩 중(`Page: init`)일 때 표시할 UI.
    - **Fields**: 이메일, 이름, 닉네임(필수), 가입일, 권한(Role).
- **Feedback (State Machine)**:
    - **Loading (`Page: init`)**: Skeleton UI 표시.
    - **Success (`Data: present`)**: 데이터 바인딩 후 화면 표시.
    - **Error 401 (`Auth: stop`)**: 로그인 페이지로 리다이렉트 (Toast: "로그인이 필요합니다").
    - **Error 404 (`Data: error`)**: "사용자 정보를 찾을 수 없습니다." 에러 표시.

**DATA SPEC**:
- **API Endpoint**: `GET /users/me`
- **Response Type (`UserDetail`)**:
  ```typescript
  {
    user_id: number;
    email: string;
    name: string;
    nickname: string; // 필수 확인
    role: string;
    created_at: string;
    // ...
  }
  ```

**IMPLEMENTATION STEPS**:
1.  **Type Definition**: `src/category/user/types.ts` 생성 및 `UserDetail` 인터페이스 정의.
2.  **API Function**: `src/category/user/user_api.ts` -> `getUserMe()` 함수 구현.
3.  **Hook**: `src/category/user/hook/use_user_me.ts` -> `useQuery` 사용.
4.  **Component**: `src/category/user/page/my_page.tsx` 구현.
5.  **Route**: `src/app/routes.tsx`에 `/user/me` 경로 추가.
6.  **Navigation**: `src/category/home/page/home_page.tsx`의 '마이 페이지' 버튼이 `/user/me`로 잘 연결되어 있는지 확인.
