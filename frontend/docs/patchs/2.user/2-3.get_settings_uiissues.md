# 🛠️ Issue Log: Phase 2-3 (Get Settings UI) Implementation

## 📋 개요
- **작업**: 사용자 환경 설정 조회 페이지(`/settings`) 구현 및 UI 컴포넌트(Switch, Select) 바인딩.
- **기간**: 2026.01.13
- **주요 성과**:
  - **Visual Settings**: 텍스트 대신 `Switch`, `Select` 컴포넌트를 사용하여 직관적인 설정 UI 구현.
  - **Read-Only State**: 조회 전용 페이지로서, 컨트롤 조작 시 실제 API 요청 없이 UI 상태만 보여주도록 처리.
  - **CLI Migration**: `shadcn-ui`에서 `shadcn` 패키지로의 CLI 마이그레이션 대응.

## 🐛 발생 이슈 및 해결 (Troubleshooting)

### 1. 진입점(Entry Point) 누락 (AI Planning Oversight)
- **현상**: `/settings` 페이지와 라우팅 로직은 완벽하게 구현했으나, 정작 메인 화면(`HomePage`)에서 해당 페이지로 이동할 수 있는 **버튼을 만들지 않음**.
- **원인**: 개별 페이지 기능 구현(`2-3.get_settings_ui.md`)에만 집중한 나머지, 전체 네비게이션 흐름(Flow) 설계를 간과함.
- **해결**:
  - `src/category/home/page/home_page.tsx`를 수정하여 로그인 유저 섹션과 Dev Route Map에 `[⚙️ 설정]` 버튼 추가.
  - **Lesson**: "페이지를 만든다"는 정의에는 "그 페이지로 가는 문을 만든다"도 반드시 포함되어야 한다.

### 2. Shadcn/Radix 의존성 미설치 (Dependency Error)
- **현상**: `Switch`, `Select` 컴포넌트를 사용하려 했으나, 기반이 되는 `@radix-ui/react-*` 모듈들이 설치되지 않아 런타임 에러 발생 가능성 존재.
- **원인**: Shadcn UI 컴포넌트는 Radix UI Primitives에 의존하는데, 이를 설치하는 CLI 명령어(`npx shadcn add ...`)를 작업 초기에 안내하지 않음.
- **해결**:
  - 터미널에서 `npx shadcn@latest add switch select label card` 명령어를 실행하여 필요한 컴포넌트와 의존성 패키지를 일괄 설치.

### 3. Shadcn CLI 패키지 변경 (Deprecation Warning)
- **현상**: 기존 명령어 `npx shadcn-ui@latest ...` 실행 시 패키지 Deprecated 경고 발생.
- **해결**:
  - 최신 가이드에 따라 `npx shadcn@latest ...` (가운데 `-ui` 제거) 명령어로 변경하여 실행.
  - 기능상 차이는 없으나, 최신 툴체인을 유지하기 위해 변경된 명령어 사용 권장.

## 💡 Key Takeaways (교훈)
1.  **Navigation First**: 새로운 페이지를 기획할 때는 **"어디서 들어갈 것인가?"**를 가장 먼저 정의해야 한다. 버튼 없는 페이지는 유령 페이지다.
2.  **Dependencies Check**: UI 라이브러리(Shadcn 등)를 사용할 때는 해당 컴포넌트가 필요로 하는 내부 의존성(Radix 등)이 설치되어 있는지 항상 체크해야 한다.
3.  **Tooling Updates**: 오픈소스 생태계는 빠르게 변한다. CLI 경고 메시지(Deprecation)를 무시하지 말고, 제안하는 최신 명령어를 따르는 것이 장기적인 유지보수에 좋다.
