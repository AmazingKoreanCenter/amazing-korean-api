# 🚀 Backlog: User Update UI Upgrade Plan (Phase 2-2+)

## 📋 개요
Phase 2-2(MVP)는 닉네임과 이름 수정에 집중했습니다.
다음 단계에서는 앞서 제외된 **자기소개(Bio)** 기능의 백엔드 연동과 **UX 디테일(유효성 검사 피드백)**을 강화합니다.

---

## 1. 백엔드 스펙 확장 (Backend Sync)

### 📝 자기소개 (Bio) 필드 추가
- **Status**: Frontend(UI 제외됨) / Backend(미구현)
- **Action**:
  - **Backend**: `users` 테이블에 `bio (TEXT, Nullable)` 컬럼 추가 및 Update DTO 수정.
  - **Frontend**: `edit_profile_page.tsx`에 `Textarea` 컴포넌트 복구 및 Zod 스키마 재정의.

### 🖼️ 프로필 이미지 업로드 연동
- **Status**: Phase 2-1 Upgrade Plan과 연결됨.
- **Action**:
  - 정보 수정 페이지 상단에 **Avatar 클릭 시 이미지 변경** UI 추가.
  - 텍스트 정보(`json`)와 이미지(`multipart/form-data`)를 분리해서 전송할지, 통합 전송할지 백엔드와 협의 필요.

---

## 2. 입력 경험 및 유효성 (UX & Validation)

### ✅ 실시간 닉네임 중복 검사 (Debounce Check)
- **현재**: '저장하기' 버튼을 눌러야만 중복 여부(409)를 알 수 있음.
- **개선**:
  - 사용자가 닉네임 입력 후 0.5초간 멈추면(`Debounce`), 백그라운드에서 `GET /users/check-nickname` API를 호출.
  - 입력창 우측에 "사용 가능한 닉네임입니다(초록색)" 또는 "이미 사용 중입니다(빨간색)" 피드백 즉시 제공.

### 💾 변경 감지 및 이탈 방지 (Dirty Check)
- **기능**: 내용을 수정하다가 저장하지 않고 '뒤로 가기'나 '메뉴 이동'을 시도할 때.
- **UX**: "작성 중인 내용이 있습니다. 저장하지 않고 나가시겠습니까?" 컨펌 모달 출력. (`form.formState.isDirty` 활용)
