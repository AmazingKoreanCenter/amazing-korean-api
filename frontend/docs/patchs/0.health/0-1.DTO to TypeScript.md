**CONVERSION RULES (변환 규칙)**:
1.  **Zod Schema 우선 정의**: 모든 DTO에 대해 Zod 스키마를 먼저 정의하십시오 (예: `signupRequestSchema`).
    - Rust `String` -> `z.string()`
    - Rust `Option<T>` -> `z.optional()`
    - Rust `Uuid` -> `z.string().uuid()`
    - Rust `chrono::NaiveDate` -> `z.string().date()` (또는 정규식 검증)
    - Rust `Vec<T>` -> `z.array(...)`
    - Rust `i32/i64` -> `z.number().int()`
2.  **Type Inference (타입 추론)**: 정의된 Zod 스키마로부터 TypeScript 타입을 추출하여 export 하십시오.
    - 예: `export type SignupRequest = z.infer<typeof signupRequestSchema>;`
3.  **Snake Case 유지**: `user_email`, `is_active` 등 모든 필드명을 그대로 유지하십시오.
4.  **Validation 반영**: Rust 코드의 `#[validate(length(min = 8))]` 같은 속성을 확인하여, Zod의 `.min(8)`, `.email()` 등으로 반영하십시오.

**OUTPUT EXAMPLE (작성 예시)**:

```typescript
// frontend/src/types/user_dto.ts
import { z } from "zod";

// --- Signup (회원가입) ---
export const signupRequestSchema = z.object({
  email: z.string().email({ message: "유효한 이메일 주소를 입력해주세요" }),
  password: z.string().min(8, { message: "비밀번호는 최소 8자 이상이어야 합니다" }),
  name: z.string().min(2),
  nickname: z.string(),
  birth: z.string().regex(/^\d{4}-\d{2}-\d{2}$/, { message: "형식: YYYY-MM-DD" }),
  gender: z.enum(["male", "female", "other"]),
  country: z.string().default("KR"),
  language: z.enum(["ko", "en", "ja", "zh"]).default("en"),
});

export type SignupRequest = z.infer<typeof signupRequestSchema>;

// --- User Response (응답) ---
export const userResponseSchema = z.object({
  user_id: z.string().uuid(),
  email: z.string().email(),
  name: z.string(),
  // ...
});
export type UserResponse = z.infer<typeof userResponseSchema>;
