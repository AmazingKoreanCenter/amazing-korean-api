# 📝 Phase 6-11 Admin Update Video Tags - Success & Design Log

## 1. 디자인 패턴 및 아키텍처 (Design Pattern & Architecture)

### ✅ 파사드(Facade) 패턴을 통한 재사용
* **전략:** `repo::admin_update_video`는 모든 필드(Video + Tag)를 수정할 수 있는 만능 함수입니다.
* **적용:** 새로운 엔드포인트(`PATCH /tags`)를 위해 별도의 SQL을 짜는 대신, **서비스 레이어에서 '태그 수정 요청'을 '전체 수정 요청' 형식으로 변환**하여 기존 함수를 호출했습니다.
* **이점:**
    1.  **로직 파편화 방지:** 수정 로직이 한곳에 집중되어 유지보수가 쉽습니다.
    2.  **Audit Trail 유지:** 태그만 수정하더라도, 재사용된 로직 덕분에 부모 테이블인 `VIDEO`의 `updated_at` 시간이 자동으로 갱신되어 "언제 마지막으로 건드렸는지" 추적이 가능해졌습니다.

## 2. 데이터 무결성 (Data Integrity)

### ✅ 입력 값 검증의 일관성
* **확인:** `video_tag_key`의 30자 길이 제한 등 DB 제약 조건을 새로운 DTO(`VideoTagUpdateReq`)에도 동일하게 적용했습니다.
* **결과:** 어떤 경로(전체 수정 vs 태그 수정)로 들어오든 데이터 무결성이 동일한 수준으로 보장됩니다.
