## 4. 교육 방법론 & 비즈니스 전략

### 4.1 교육 방법론 핵심

- **500 핵심 문장**: 교과서/신문/TOPIK 기출에서 선별, 활용 포함 시 ~700문장
- **목표**: 습득(Acquisition) = 최종 목표, 학습(Learning) = 과정
- **패턴 변형(Pattern Variation)**: 단순 반복이 아닌, 어미·조사 변형을 통한 확장
- **자국어 번역**: 의미 이해를 위한 보조 수단 (자국어 중심 교육 아님)
- **TOPIK/EPS-TOPIK 대비**: 기출문제 유형별·등급별 분석

**3단계 교재 체계**:
| 종류 | 용도 | 대상 |
|------|------|------|
| 교사용(Teacher's Guide) | 교사가 학생을 가르칠 때 사용 | B2B 기관/교사 |
| 학생용(Workbook) | 교사가 가르칠 때 학생이 사용 | B2B 연습 교재 |
| 해설용(자습용) | 번역+해설 포함, 개인 학습 | B2C 학습자 |

### 4.2 검증된 성과 (베트남)

- **1,000명+ 학습자** 실적
- **평균 TOPIK 3-4급 달성: 200시간** (어학당 대비 3~4배 빠름)
- **TOPIK 6급: 1년 이내** 달성 사례

**200시간 달성이 가능한 이유**:
1. TOPIK 구조적 약점: 읽기+듣기만 측정 (말하기 없음) → 수용적 능력만 훈련하면 점수 달성 가능
2. Korea Herald: "Korean proficiency test measures memory, not skill"
3. 어학당 비효율: 학위과정 유학생 중 TOPIK 4급+ 보유자 **34%에 불과** (1년 이상 수강해도)
4. 어학당 구조적 문제: 다국적 교실에서 한국어만 사용 → L1 활용 불가, 범용 커리큘럼 → L1 간섭 미대응

**방법론 비판적 검증**:
| 항목 | 이전 비판 | 검증 결과 |
|------|----------|----------|
| 500문장 부족 | 영어 기준(Nation 6,000~9,000) 적용 | ❌ 한국어는 교착어 — 어간 1개 → ~60 활용형, 500 어간 → ~30,000 표면형 |
| 습득 개념 오용 | Krashen 학술 정의와 불일치 | ❌ 실무적 구분(학습=과정, 습득=목표)은 합리적 |
| CLT/TBLT 필요 | 의사소통 중심 교육 병행 제안 | ❌ 교착어 연구: Form-focused approach가 agglutinative languages에 더 효과적 |
| 200시간 TOPIK 3-4 | 비현실적 주장? | ✅ SNU 기준 600~800시간의 3~4배 빠름 — 설명 가능 (위 참조) |

### 4.3 22개 언어 확장 설계 원칙

- **커리큘럼은 한국어 하나** — 22개 언어 모두 동일한 핵심 콘텐츠
- **자국어는 보조 수단** — 번역+설명만 자국어, 학습 자체는 한국어 중심
- **한국어 고유 문법/발음** → 보편적 대상·표현으로 설명 + 한국적 요소 추가
- L1별 커스터마이징 = **번역+설명 레이어에만 한정** → 스케일링 현실적
- 오답률/오류 데이터는 운영하면서 점진적으로 수집 → 급하지 않음

### 4.4 교재 판매 채널 전략

| 채널 | 상품 | 이름 | 대상 |
|------|------|------|------|
| **외부 플랫폼** (Amazon, Google Play, Apple Books 등) | 1종만 | **Amazing Korean 1** (정식 교재명) | B2C 개인 학습자 |
| **자체 웹/앱** | 3종 | Amazing Korean 1 + Teacher's Guide + Workbook | B2B 교사/기관 + B2C 심화 |

**네이밍 원칙**:
- 외부 플랫폼: **절대 "해설용/부록/가이드" 붙이지 않음** → 불완전한 상품으로 인식되어 전환율 하락
- 자체 채널: Teacher's Guide / Workbook = 부가가치 상품 → 자체 채널 유입 동기 부여
- 참고: TTMIK은 외부에서 "TTMIK Korean 1"이지 "해설판"이 아님

### 4.5 비즈니스 모델

**3단계 상품 라인업**:
1. **한국어 기초** (현재) — 22개 언어, 경쟁자 전무
2. **TOPIK I/II** — 다국어 시장 공백, 기초→TOPIK 자연스러운 학습 경로
3. **EPS-TOPIK** — 완전한 블루오션, 16개 송출국

**5단계 판매 채널**:
1. 자체 사이트 (Paddle) — 수수료 5%+$0.50, 전 세계, 3종 전체
2. Google Play Books — 60개국+, 전 언어
3. 교보문고 / Yes24 — 국내
4. Apple Books — 50개국+
5. Amazon KDP / Kobo — 영어/일본어 등

**3대 고객층**:
1. **B2C 개인 학습자**: 외부 플랫폼 + 자체사이트 (정식 교재 1종)
2. **B2B 교사/기관**: 자체사이트 전용 (Teacher's Guide + Workbook)
3. **EPS-TOPIK 지원자**: Google Play + 자체사이트 (저가 $5~10)

**핵심 경쟁 우위: "22개 언어"**:
- 단순 차별화가 아닌 **진입 장벽(moat)** — 후발 주자가 동일 규모로 추격하려면 22개 언어 × 번역 비용 × 교재 품질 유지
- **수평 확장(언어)** + **수직 확장(기초→TOPIK→EPS-TOPIK→실전)** 동시 성장

**1차/2차 목표**:
- 1차: TOPIK/EPS-TOPIK 점수 달성 — 현재 집중
- 2차: 실전 한국어 능력(말하기/쓰기) — 로드맵에 포함, 추후 구체화

### 5.3 플랫폼별 결제 전략

#### Apple (iOS) — 기본 결제

- **StoreKit 2** 사용 (iOS 15+, async/await, JWS 서명 트랜잭션)
- **App Store Server Notifications V2** (JSON 기반 웹훅)
- **Small Business Program** 등록 → 15% 수수료
- **무료 체험**: 7일 (introductory offer)
- **미국**: External Purchase Link 엔티틀먼트 → 웹 결제 링크 허용 (Apple 27% 부과)
- **EU (DMA)**: 대안 결제 수단 허용, Core Technology Fee 적용 (100만 설치 이하 면제)
- **Family Sharing**: 최대 5명, 교육 앱에 강점 ("온 가족이 한국어 학습")

#### Google (Android) — 기본 결제

- **Play Billing Library 7.x** 사용
- **Real-Time Developer Notifications (RTDN)** via Cloud Pub/Sub
- 구독 전체 수수료 **15%** (매출 무관)
- **User Choice Billing**: EU/인도/호주/일본 등에서 대체 결제 시 4% 추가 할인 (11%)
- **미국**: 2025년 말부터 대체 결제 허용 (수수료 미정)
- **구독 일시정지**: Google Play만 지원 (Apple 미지원) — 이탈 방지에 효과적
- **Grace Period**: 7일 권장, 비자발적 이탈 ~30% 회복

#### Paddle (Web) — 대안 결제

- **이미 구현 완료** (Phase 1-5)
- 가장 낮은 수수료 (5% + 결제 수수료)
- **MoR (Merchant of Record)**: 세금/VAT 자동 처리
- 웹에서 구독 → iOS/Android 앱에서 동일 계정으로 이용
- 마케팅 이메일/웹사이트에서 직접 결제 유도 가능

### 5.4 통합 구독 관리

#### DB 스키마 확장

```sql
-- 기존 payment_provider enum 확장
ALTER TYPE payment_provider ADD VALUE 'apple';
ALTER TYPE payment_provider ADD VALUE 'google';

-- 기존 subscriptions 테이블에 provider별 ID 추가
ALTER TABLE subscriptions
    ADD COLUMN apple_original_transaction_id TEXT,
    ADD COLUMN google_purchase_token TEXT;
    -- paddle_subscription_id 기존 컬럼 유지

-- Provider별 UNIQUE 인덱스
CREATE UNIQUE INDEX idx_sub_apple_txn ON subscriptions(apple_original_transaction_id)
    WHERE apple_original_transaction_id IS NOT NULL;
CREATE UNIQUE INDEX idx_sub_google_token ON subscriptions(google_purchase_token)
    WHERE google_purchase_token IS NOT NULL;
```

#### Webhook 이벤트 매핑

| 내부 이벤트 | Apple (ASN V2) | Google (RTDN) | Paddle |
|-------------|----------------|---------------|--------|
| 구독 생성 | `SUBSCRIBED` | `SUBSCRIPTION_PURCHASED` | `subscription.created` |
| 갱신 | `DID_RENEW` | `SUBSCRIPTION_RENEWED` | `subscription.updated` |
| 취소 요청 | `DID_CHANGE_RENEWAL_STATUS` | `SUBSCRIPTION_CANCELED` | `subscription.canceled` |
| 만료 | `EXPIRED` | `SUBSCRIPTION_EXPIRED` | status → canceled |
| 일시정지 | N/A (미지원) | `SUBSCRIPTION_PAUSED` | `subscription.paused` |
| Grace Period | `DID_FAIL_TO_RENEW` | `SUBSCRIPTION_IN_GRACE_PERIOD` | `subscription.past_due` |
| 환불 | `REFUND` | `SUBSCRIPTION_REVOKED` | `transaction.refunded` |

#### 엔티틀먼트 확인 패턴

```
모든 플랫폼 (iOS, Android, Web)
    → 서버에 entitlement 확인 요청 (JWT 인증)
    → subscriptions 테이블 조회 (provider 무관, user_id 기준)
    → 활성 구독 여부 반환

원칙: 앱은 Apple/Google에 직접 구독 상태를 확인하지 않음.
      서버가 유일한 진실의 원천 (SSoT), 웹훅으로 업데이트.
```

### 5.5 Reader App 예외 해당 여부

**교육 앱은 Reader App에 해당하지 않음.**

Reader App (Netflix, Kindle, Spotify)은 "이미 구매한 콘텐츠에 접근"하는 앱.
한국어 학습 앱은 "서비스 자체를 제공"하므로 IAP 의무.

→ iOS에서는 Apple IAP를 반드시 제공해야 함 (미국 외부 링크 허용은 별도)
