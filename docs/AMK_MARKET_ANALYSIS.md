### 4.3 플랫폼별 결제 전략

#### Apple (iOS) — 기본 결제

- **StoreKit 2** 사용 (iOS 15+, async/await, JWS 서명 트랜잭션)
- **App Store Server Notifications V2** (JSON 기반 웹훅)
- **Small Business Program** 등록 → 15% 수수료
- **무료 체험**: 7일 (introductory offer)
- **미국**: External Purchase Link 엔티틀먼트 → 웹 결제 링크 허용 (Apple 27% 부과)
- **EU (DMA)**: 대안 결제 수단 허용, Core Technology Fee 적용 (100만 설치 이하 면제)
- **Family Sharing**: 최대 5명, 교육 앱에 강점 ("온 가족이 한국어 학습")

#### Google (Android) — 기본 결제

- **Play Billing Library 7.x** 사용
- **Real-Time Developer Notifications (RTDN)** via Cloud Pub/Sub
- 구독 전체 수수료 **15%** (매출 무관)
- **User Choice Billing**: EU/인도/호주/일본 등에서 대체 결제 시 4% 추가 할인 (11%)
- **미국**: 2025년 말부터 대체 결제 허용 (수수료 미정)
- **구독 일시정지**: Google Play만 지원 (Apple 미지원) — 이탈 방지에 효과적
- **Grace Period**: 7일 권장, 비자발적 이탈 ~30% 회복

#### Paddle (Web) — 대안 결제

- **이미 구현 완료** (Phase 1-5)
- 가장 낮은 수수료 (5% + 결제 수수료)
- **MoR (Merchant of Record)**: 세금/VAT 자동 처리
- 웹에서 구독 → iOS/Android 앱에서 동일 계정으로 이용
- 마케팅 이메일/웹사이트에서 직접 결제 유도 가능

### 4.4 통합 구독 관리

#### DB 스키마 확장

```sql
-- 기존 payment_provider enum 확장
ALTER TYPE payment_provider ADD VALUE 'apple';
ALTER TYPE payment_provider ADD VALUE 'google';

-- 기존 subscriptions 테이블에 provider별 ID 추가
ALTER TABLE subscriptions
    ADD COLUMN apple_original_transaction_id TEXT,
    ADD COLUMN google_purchase_token TEXT;
    -- paddle_subscription_id 기존 컬럼 유지

-- Provider별 UNIQUE 인덱스
CREATE UNIQUE INDEX idx_sub_apple_txn ON subscriptions(apple_original_transaction_id)
    WHERE apple_original_transaction_id IS NOT NULL;
CREATE UNIQUE INDEX idx_sub_google_token ON subscriptions(google_purchase_token)
    WHERE google_purchase_token IS NOT NULL;
```

#### Webhook 이벤트 매핑

| 내부 이벤트 | Apple (ASN V2) | Google (RTDN) | Paddle |
|-------------|----------------|---------------|--------|
| 구독 생성 | `SUBSCRIBED` | `SUBSCRIPTION_PURCHASED` | `subscription.created` |
| 갱신 | `DID_RENEW` | `SUBSCRIPTION_RENEWED` | `subscription.updated` |
| 취소 요청 | `DID_CHANGE_RENEWAL_STATUS` | `SUBSCRIPTION_CANCELED` | `subscription.canceled` |
| 만료 | `EXPIRED` | `SUBSCRIPTION_EXPIRED` | status → canceled |
| 일시정지 | N/A (미지원) | `SUBSCRIPTION_PAUSED` | `subscription.paused` |
| Grace Period | `DID_FAIL_TO_RENEW` | `SUBSCRIPTION_IN_GRACE_PERIOD` | `subscription.past_due` |
| 환불 | `REFUND` | `SUBSCRIPTION_REVOKED` | `transaction.refunded` |

#### 엔티틀먼트 확인 패턴

```
모든 플랫폼 (iOS, Android, Web)
    → 서버에 entitlement 확인 요청 (JWT 인증)
    → subscriptions 테이블 조회 (provider 무관, user_id 기준)
    → 활성 구독 여부 반환

원칙: 앱은 Apple/Google에 직접 구독 상태를 확인하지 않음.
      서버가 유일한 진실의 원천 (SSoT), 웹훅으로 업데이트.
```

### 4.5 Reader App 예외 해당 여부

**교육 앱은 Reader App에 해당하지 않음.**

Reader App (Netflix, Kindle, Spotify)은 "이미 구매한 콘텐츠에 접근"하는 앱.
한국어 학습 앱은 "서비스 자체를 제공"하므로 IAP 의무.

→ iOS에서는 Apple IAP를 반드시 제공해야 함 (미국 외부 링크 허용은 별도)

## 5. 수익 최적화

### 5.1 가격 전략 재검토

#### 현재 가격 vs 권장 가격

| 플랜 | 현재 | 권장 | 월 환산 | 할인율 |
|------|------|------|---------|--------|
| 1개월 | $10 | **$12.99** | $12.99 | - |
| 3개월 | $25 | **$29.99** | $10.00 | 23% |
| 6개월 | $50 | **$47.99** | $8.00 | 38% |
| 12개월 | $100 | **$79.99** | $6.67 | **49%** |

**근거**:
- 앵커 프라이싱: 높은 월간 가격($12.99)이 장기 플랜의 가치를 부각
- Duolingo: $12.99/월 또는 $83.99/년 (46% 할인) — 경쟁력 유지
- 현재 6개월/12개월의 월환산이 동일($8.33) → 12개월 선택 유인 없음
