> **작성일**: 2026-03-13
> **인쇄소**: 에이프린트 (aprint1.com)
> **제품**: Amazing Korean 교재 22개 언어 × 2에디션(교사/학생) = 44세트
> **사양**: A4, 124p, 표지 스노우지 250g(무광 코팅), 내지 백색 모조 100g, 좌철 **일반 무선제본**

## 1. 에이프린트 작업가이드 요구사항 원문 요약

### 1.1 색상 (01)

- 작업 템플릿은 반드시 인쇄모드인 **CMYK**로 작업 (RGB모드는 인쇄용 모드가 아님)
- 별색(DIC, PANTONE 등) 사용 후 CMYK 변환 시 화면색상과 다르게 탁하고 진하게 나옴
- 모니터(RGB) ≠ 인쇄(CMYK) 색상 차이 발생
- CMYK 4색 혼합 시 오차 범위 커짐
- 같은 색상값 편차 최소 40~50% 권장. 20%는 옵셋에서 표현 안됨
- 오버프린트는 당사에서 별도 확인하지 않음 (고객 확인 후 접수)
- **글자색: CMYK 혼합 4색 사용 금지 / K100 사용**

### 1.2 서체 (02)

- 글자크기 최소 **6~7pt** 이상
- **모든 서체는 반드시 윤곽선만들기나 곡선으로 변환 (Create Outlines)**
- 변환하지 않은 서체 사용으로 발생된 인쇄오류는 당사 책임 없음
- 출력불가 서체: HJ, AppleGothic, AppleMyungjo, Seoul, 한강, 문조, 국민서체
- 약물/한자 사용 시 지원 서체 확인 필수
- **PDF 변환 시 반드시 윤곽선만들기 후 변환**
- **아크로뱃 > 파일 > 속성(Ctrl+D) 문서 속성에서 글꼴이 나타나지 않아야 함**

### 1.3 효과 (03)

- 그라데이션, 투명도 및 각종 효과 → 반드시 그림화
- 서체 제외 이미지는 레스터화 또는 비트맵처리
- 레스터화: CMYK, 300dpi, 배경 흰색/투명
- 모든 프로그램에서 레이어 사용 시 반드시 병합 후 그룹화

### 1.4 포토샵 주의사항 (04)

- 모든 이미지는 CMYK로 변환
- 원본 72dpi를 300dpi로 바꿔도 선명해지지 않음 (원본부터 300dpi)

### 1.5 PDF 저장방법

- Adobe PDF 사전 설정: 출판 품질 선택
- 도련 상하좌우 5mm 설정
- 모든 서체 아웃라인 처리 후 변환
- 클리핑마스크 처리, 별색→CMYK 변환, 이미지링크 확인

### 1.6 무선제본 작업가이드

- 표지와 내지 **별도 작업** (세네카 때문)
- **표지**: 스프레드 형태 (앞표지 + 세네카 + 뒷표지)
- 견적창에서 재단사이즈 입력 시 세네카 두께 자동 계산 → 표지 작업사이즈 표시
- **내지**: 펼침면이 아닌 **1페이지(낱장) 사이즈**로 작업
- **작업사이즈** = 재단사이즈 + 사방 2mm (가로/세로 총 +4mm)
- A4 기준: 재단 210×297mm → 작업 **214×301mm**

### 1.7 PUR제본 세네카 규정

- **PUR제본 양면인쇄 표지**에만 해당 (일반 무선제본은 해당 없음)
- "세네카(책등) 부분에는 인쇄가 되지 않도록 흰 여백으로 처리"
- **세네카 폭 + 좌·우 각각 5mm의 흰 여백** 필요
- 이유: PUR 접착제 특성상 인쇄면 접착력 저하

### 1.8 재단 (08)

- 재단 시 밀림 현상 **±1~2mm** 발생 가능
- 테두리/띠 작업은 재단선에서 **3mm 안쪽**으로 작업
- 진한 바탕색 인쇄물은 뒷묻음/뜯김 현상 발생 가능 (재작업 불가)

### 1.9 기타 (09)

- 선 최소 굵기 **0.5pt**
- 가는 선에 4원색 사용 시 핀 어긋남 가능
- CMYK 색상값 합 **250% 이상** 시 뒷묻음 발생 가능
- 인쇄 시 10% 미만 색상 오차 발생 가능
- 모든 인쇄물은 시안확정 필요 (4시 이전 시안확정+결제 완료 시 작업 진행)

### 1.10 PDF파일 만들기 전 주의사항 (이미지 가이드)

- ① 모든 서체는 반드시 윤곽선만들기나 곡선으로 변환 (Create Outlines)
- ② 그라데이션, 패턴, 투명도 및 각종 효과 → 반드시 그림화 (레스터화 또는 비트맵처리)
- ③ 기타 자세한 내용은 공통주의사항 참조

### 1.11 작업 프로그램에서 PDF만들기 (이미지 가이드)

- PDF작업 파일은 앞면, 뒷면을 따로 분리하지 않음
- **앞면과 뒷면을 순서대로 페이지로 만들어 한 개의 파일로 접수**

## 2. 우리 교재 PDF 현황

### 2.1 생성 도구

- **Puppeteer** (Chromium headless) → HTML → PDF
- Puppeteer는 **DeviceRGB**로만 PDF 생성 (CMYK 출력 불가)
- `preferCSSPageSize: true` → CSS `@page` 사이즈 따름

### 2.2 내지 PDF 사양

| 항목 | 값 |
|------|-----|
| 페이지 수 | 124p |
| 페이지 사이즈 | 209.889mm × 297.011mm (≒ 210×297mm) |
| 전 페이지 동일 | ✓ (124p 전수 검증) |
| ColorSpace | DeviceRGB (2건) + DeviceGray (174건) |
| @page CSS | `size: 210mm 297mm; margin: 0;` |

### 2.3 내지 폰트 현황

| 폰트 | 타입 | 임베딩 | 아웃라인 | 비고 |
|------|------|--------|---------|------|
| NotoSansKR-Thin | **Type 3** (벡터 패스) | ✓ sub | **사실상 아웃라인** | ~546개 (전 페이지) |
| Lato-Regular | CID TrueType | ✓ sub | **✗ 아웃라인 아님** | 영문 본문 폰트 (CSS 지정) |
| Lato-Bold | CID TrueType | ✓ sub | **✗ 아웃라인 아님** | 영문 볼드 |
| Lato-Black | CID TrueType | ✓ sub | **✗ 아웃라인 아님** | 영문 블랙 |
| DejaVuSans | CID TrueType | ✓ sub | **✗ 아웃라인 아님** | Chromium 특수문자 폴백 (CSS 미지정) |
| DejaVuSans-Bold | CID TrueType | ✓ sub | **✗ 아웃라인 아님** | Chromium 특수문자 폴백 |

- **CID TrueType 5종은 전 22개 언어 동일** (VN, MY, TH, EN 교차 검증 완료)
- 언어별로 해당 스크립트용 Noto Sans 폰트(Type 3)가 추가될 수 있음 (예: MY → NotoSansMyanmar-Regular Type 3). B-3 아웃라인 대상 아님
- Type 3: Chromium이 글리프를 벡터 패스로 변환 (Create Outlines와 동등)
- CID TrueType: 폰트 서브셋 임베딩 (아웃라인 처리 아님)

### 2.4 표지 PDF 사양

| 항목 | 값 |
|------|-----|
| 페이지 수 | 2p (p1: 외면 스프레드, p2: 내면 스프레드) |
| 페이지 사이즈 | 431.46mm × 300.91mm (≒ 431.5mm × 301mm) |
| 스프레드 구성 | 좌도련(2) + 뒷표지(210) + 세네카(7.56) + 앞표지(210) + 우도련(2) |
| ColorSpace | DeviceGray (3건) + Indexed DeviceRGB |

### 2.5 표지 폰트 현황

| 구분 | 폰트 | 타입 | 아웃라인 | 비고 |
|------|------|------|---------|------|
| **ISBN 있는 역본** (20개) | NotoSansKR, Inter | Type 3 | ✓ | |
| | **Gulim** | CID TrueType | **✗** | nl.go.kr 바코드 PDF에서 pdf-lib embedPdf로 유입 |
| | **ArialMT** | CID TrueType | **✗** | 동일 원인 |
| **ISBN 없는 역본** (24개) | NotoSansKR, Inter | Type 3 | ✓ | 바코드 없음 → CID TrueType 없음 |

### 2.6 최소 폰트 사이즈

| 사이즈 | 사용처 | 파일 |
|--------|-------|------|
| **6.5pt** | `.struct-table .cell-translation` (구조 테이블 번역 텍스트) | table.css:554 |
| **6.8pt** | `.formal-table .sub-header th` + `.formal-table td` (격식체 테이블) | table.css:857, 871 |
| 7.5pt | 기타 보조 텍스트 | — |

→ 에이프린트 기준 6~7pt 이상 → **6.5pt는 기준 하한선에 위치 (통과)**

### 2.7 선 굵기

| 굵기 | pt 환산 | 사용처 | 색상 | CMYK 합산 |
|------|---------|-------|------|-----------|
| 0.3mm | **0.85pt** | 테이블 테두리 (table.css) | #e2e8f0 | 15% |
| 0.3mm | **0.85pt** | 격식체 테이블 테두리 | #e2e8f0, #d1d5db | 15~20% |
| 1px | **0.75pt** | 페이지 푸터 구분선, 문장 구분선 등 | #e2e8f0 | 15% |
| 1px | **0.75pt** | 기본 테이블 셀 테두리 | #cbd5e1, #94a3b8 | 20~59% |

→ 에이프린트 기준 최소 0.5pt → **모두 0.75pt 이상 (통과)**
→ 가는 선(1px)의 색상이 모두 연한 회색이라 4색 혼합 핀 어긋남 위험 낮음

## 5. 처리 완료 요약

### 5.1 전체 처리 항목

| 항목 | 상태 | 처리 방법 |
|------|:----:|----------|
| **B-1** RGB→CMYK | ✅ 완료 | Ghostscript `-sColorConversionStrategy=CMYK` |
| **B-2** K100 텍스트 | ✅ 완료 | Ghostscript CMYK 변환 시 자동 K100 매핑 |
| **B-3** 폰트 아웃라인 | ✅ 완료 | Ghostscript `-dNoOutputFonts` |
| **B-4** 내지 사이즈 | ✅ 해당 없음 | 에이프린트 정사이즈 접수 가능 |
| **B-5** 스프레드 사이즈 | ✅ 허용 범위 | 0.06mm 차이 (도련 2mm 내 흡수) |
| **§3.3** PUR 세네카 | ✅ 해당 없음 | 일반 무선제본 확정 |
| **§3.4** 내면 세네카 5mm | ✅ 확인 | 현재 패딩 충분 (최소 12mm) |
| **CMYK 색상 구분** | ✅ 완료 | 3색 대체 (아래 §3.5 참조) |

### 5.2 Ghostscript 일괄 변환 명령어 (B-1 + B-2 + B-3)

```bash
gs -dNOPAUSE -dBATCH -sDEVICE=pdfwrite \
   -sColorConversionStrategy=CMYK \
   -dProcessColorModel=/DeviceCMYK \
   -dNoOutputFonts \
   -sOutputFile=output_cmyk.pdf \
   input_rgb.pdf
```

일괄 변환 스크립트: `scripts/textbook/convert_print.sh`
