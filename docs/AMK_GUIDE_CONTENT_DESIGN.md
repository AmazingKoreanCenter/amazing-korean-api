## §2 설계 원리

1. **모든 것은 블록** — .txt 세그먼트 스트림을 `guide_block` 단일 테이블에 저장. 단원 도입·표 셀·어휘·노트 전부 한 테이블, `block_seq` 하나로 페이지 순서 완결. 새 콘텐츠 유형 = enum 값 추가.
2. **문장(500)만 학습항목으로 승격** — `guide_sentence` = 학습 기록·활동·향후 간격복습(FSRS)의 FK 대상. 텍스트는 section 블록 1곳에만(정규화). HTML이 한 문장을 페이지 내 6곳에 복제하던 것을 해소.
3. **번역 = content_translations 재사용** — content_type 값 2개 추가만. 번역 시점 `source_version` 기록으로 편집 후 stale 자동 판정.
4. **활동은 코드** — 복습 4종(Read Along/플래시카드/매칭/Writing Test)·쓰기연습은 sentence 파생 렌더. 앱 UI 문자열(Show Answer 등)은 UI i18n(콘텐츠 번역 아님).

## §3 스키마 (PR-1 마이그레이션)

테이블 5 + enum 4 + content_translations 확장. DDL 전문:

```sql
-- 단원 (67행)
CREATE TYPE guide_state_enum    AS ENUM ('ready','open','close');
CREATE TYPE guide_category_enum AS ENUM ('sentence_structure','predicate','adverbial','misc');

CREATE TABLE guide (
  guide_id           BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  guide_idx          VARCHAR(100) UNIQUE NOT NULL,      -- 'guidev2-05' ~ 'guidev2-71'
  guide_seq          INT NOT NULL,
  guide_state        guide_state_enum NOT NULL DEFAULT 'ready',   -- ready=숨김 (study 선례)
  guide_category     guide_category_enum NOT NULL,
  guide_source       VARCHAR(50) NOT NULL DEFAULT 'guide500',     -- 미래 콘텐츠군(한글 트랙 등) 수용
  sentence_start     INT,
  sentence_end       INT,
  title_ko VARCHAR(200),  title_en VARCHAR(200),
  subtitle_ko VARCHAR(300), subtitle_en VARCHAR(300),
  source_version     INT NOT NULL DEFAULT 1,
  updated_by_user_id BIGINT,
  guide_created_at   TIMESTAMPTZ NOT NULL DEFAULT now(),
  guide_updated_at   TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- 콘텐츠 블록 (~11,300행: 세그먼트 전체 — 번역 대상 + ko학습칸/기호/빈칸 포함)
CREATE TYPE guide_block_type_enum AS ENUM (
  'title','paragraph','pattern','section','vocab','ui','note',
  'blockquote','other','table_header','table_cell','practice');

CREATE TABLE guide_block (
  guide_block_id     BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  guide_id           BIGINT NOT NULL REFERENCES guide(guide_id) ON DELETE CASCADE,
  block_seq          INT NOT NULL,            -- .txt MMM 순서 승계 ×10 (편집 삽입 여유)
  block_type         guide_block_type_enum NOT NULL,
  sentence_no        INT,                     -- 문장 귀속 논리 키 (1~500, NULL=단원 레벨, FK 아님)
  text_ko TEXT,  text_en TEXT,
  marker             VARCHAR(40),             -- symbol_only / table_content_ko_learning / content_ko_learning / table_content_en / empty:짝 / empty:헤더
  table_no INT, row_no INT, col_no INT,       -- 표 좌표 Tn_Rr_Cc (비표 블록 NULL)
  col_span INT, row_span INT,                 -- 병합 메타 (NULL=1, HTML 1회 추출)
  meta               JSONB,                   -- 렌더 힌트·확장 예비
  source_version     INT NOT NULL DEFAULT 1,  -- admin 편집(text_ko/text_en 변경) 시 +1
  legacy_key         VARCHAR(50) UNIQUE,      -- 'guidev2:05_001' (초기 시딩분만)
  updated_by_user_id BIGINT,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE (guide_id, block_seq)
);

-- 문장 = 학습항목 오버레이 (500행, 텍스트는 section 블록 보유)
CREATE TABLE guide_sentence (
  guide_sentence_id  BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  guide_id           BIGINT NOT NULL REFERENCES guide(guide_id),
  sentence_no        INT UNIQUE NOT NULL,     -- 1~500 전역
  guide_block_id     BIGINT UNIQUE NOT NULL REFERENCES guide_block(guide_block_id),
  pron_ko            TEXT,                    -- 발음형 (후속 rule-based G2P 백필)
  speech_level       VARCHAR(30),             -- 화계 (값 체계 확정 전 nullable — 주체높임과 2축 분리 원칙)
  subject_honorific  BOOLEAN,
  audio_url          TEXT,                    -- 후속 TTS 자산 (D-6)
  updated_by_user_id BIGINT,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- 학습 기록 (study_task_log/status 패턴)
CREATE TYPE guide_activity_enum   AS ENUM ('sentence_write','read_along','flashcard','matching','writing_test');
CREATE TYPE guide_log_action_enum AS ENUM ('view','attempt','correct','wrong','reveal','complete');

CREATE TABLE guide_sentence_log (
  guide_sentence_log_id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  user_id            BIGINT NOT NULL,
  guide_sentence_id  BIGINT NOT NULL REFERENCES guide_sentence(guide_sentence_id),
  activity           guide_activity_enum NOT NULL,
  action             guide_log_action_enum NOT NULL,
  answer             JSONB,
  created_at         TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE guide_sentence_status (
  user_id            BIGINT NOT NULL,
  guide_sentence_id  BIGINT NOT NULL REFERENCES guide_sentence(guide_sentence_id),
  try_count          INT NOT NULL DEFAULT 0,
  is_solved          BOOLEAN NOT NULL DEFAULT false,
  last_attempt_at    TIMESTAMPTZ,
  PRIMARY KEY (user_id, guide_sentence_id)
);

-- 번역 연결 (새 테이블 없음)
ALTER TYPE content_type_enum ADD VALUE 'guide';        -- field_name: 'title'/'subtitle'
ALTER TYPE content_type_enum ADD VALUE 'guide_block';  -- field_name: 'text'
ALTER TABLE content_translations ADD COLUMN source_version INT;  -- 번역 기준 원문 버전 (기존 행 NULL)
```

**stale 판정**: `content_translations.source_version < guide_block.source_version` → admin 대시보드 노출 + 디프 export(맥미니 재번역) 대상.

**학습 연구 반영**(`AMK_CONTENT_SEGMENT_SPEC §7.5~7.7`): 한 스텝=한 문장 청킹·시도-먼저 = 뷰어 흐름 / 항목별 학습상태·provenance = guide_sentence + log·status / 화계 2축·발음형·오디오 = sentence 컬럼 4개(자리 확보, 값 후속) / FSRS·페이딩·인터리빙 = Phase B에 `(user_id, guide_sentence_id)` 키 테이블 추가만으로 수용(스키마 변경 0).

**의도된 트레이드오프**: ① 역할 라벨("Subject/주어")을 셀 그대로 둠(enum 정규화 안 함, 간결 우선) — 라벨 일괄 수정 시 해당 번역 일괄 stale 수용. ② ko/en이 도메인 컬럼 — ko 수정 시 en(36언어 원천)도 함께 수정 필요, admin 화면에 ko/en 병렬 배치로 대응.

## §4 초기 시딩 (1회성 부트스트랩)

1. **변환 스크립트**(Python, 1회): `해설집/*.txt` 67(전 세그먼트 — 번역 제외분 포함) + HTML 67(**병합 메타 추출** + **드리프트 전수 대조 리포트**) + `guide_manifest.json`(단원↔문장범위↔카테고리) → `guide_seed.json`. en 표시 텍스트는 `1_en_source_reference.json` 복원본({{koN}} 복원 완료, 673건/1,405개 누락 0 검증 — §6).
2. **`src/bin/seed_guide.rs`**:
   - `--input guide_seed.json`: guide/guide_block/guide_sentence upsert (멱등 키 = guide_idx / legacy_key).
   - `--translations <file>`: zh_cn·id → content_translations. **{{koN}} 복원 후 완성문 저장**, 트레일링 `},` 결함 2건(`guidev2:18_002` zh, `guidev2:24_234` id) 제거 규칙 내장, **status='approved'** 적재(노출 의도 + 맥미니 검증 통과분), source_version=1.
   - **재실행 가드**: 편집 흔적(updated_by_user_id NOT NULL 행 존재) 시 거부 — DB가 SoT가 된 후 시드가 편집을 덮어쓰는 사고 차단 (explanation 선례의 멱등 재시딩 관행은 본 도메인에서 **금지**로 역전).
3. **prod 전달**: 번역 4파일 + guide_seed.json = **커밋 금지(콘텐츠 유출 방지)** → Docker COPY 불가 → **scp 1회 수동 전송** 후 `docker exec` 시딩. (선례 docker exec 절차 + scp 단계, DEPLOY_OPS 갱신은 PR-1에서)

## §8 프론트 UI 계획 (PR-2/PR-3 상세)

- **모듈**: `frontend/src/category/guide/` (types → api → hook → page 컨벤션).
- **페이지 2 + admin**: ① 단원 목록(카테고리 4색 테마: sentence 파랑/predicate 보라/adverbial 초록/misc 앰버 — HTML 헤더 그라데이션 승계) ② 단원 학습 페이지(아래) ③ admin 편집(PR-3, admin_study_detail 인라인 편집 패턴).
- **단원 학습 페이지 = HTML 동등 재현**: 이중언어 헤더 → 문법 도입부(블록 타입별 렌더러: gstep 배지, pattern 다이어그램, 활용 도출표=표 격자, note/비교표) → 문장 사이클 ×N(영어 문장 → 어휘 토글 → 쓰기 textarea 즉시채점 → 정답 토글+TTS → 해설표) → 복습 4종(Read Along / 플래시카드 3D flip / 매칭 게임 / Writing Test 일괄채점) → 축하 모달.
- **이식 로직**: `speakKorean`(Web Speech API, ko-KR rate 0.85, 보이스 우선순위 Yuna→SunHi→Heami→Google) + `wtNormalize` 채점(공백·문장부호 제거 완전일치). 기존 `ExplanationSection` 렌더러의 block_type 추가식 구조 재사용.
- **HTML 대비 신규**: 진행 저장(guide_sentence_status — HTML은 localStorage조차 없음), 36언어 콘텐츠 표시(getContentLang → ?lang=, 기존 콘텐츠 언어/UI 언어 분리 인프라 재사용), 로그인 연동.
- **스타일**: HTML 인라인 CSS를 그대로 복사하지 않고 기존 디자인 시스템(shadcn/ui + AMK_DESIGN_SYSTEM 토큰)으로 재구현. 레이아웃 구조·학습 흐름은 HTML 준수. 모바일 반응형(HTML @media 의도 승계).
- **앱 UI 문자열**(Show Answer/placeholder/게임 안내문/축하 문구): 기존 UI i18n(i18next) — 콘텐츠 번역과 분리(세그먼트 스펙 §8 #4 해소).
